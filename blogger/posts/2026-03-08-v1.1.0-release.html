v1.1.0 Release Notes

Claude Bridge v1.1.0 brings interactive budget management, two-phase task orchestration, and a critical security fix for tool permissions.

New: /budget — Runtime Budget Management

Daily budget is now fully manageable from Telegram — no config file edits, no service restarts.

/budget

Shows current budget status with InlineKeyboard buttons:

  • Turn On / Turn Off — toggle budget enforcement
  • Set Amount — type a number to set a new daily limit (e.g. 50, 200)

Budget settings are persisted to SQLite. Changes take effect immediately on the next message — the bot reads from the database on every request, not from an in-memory constant.

Default budget raised from $5 to $100, reflecting realistic usage with Claude Max subscriptions.

New: /task — Two-Phase Task Orchestration

Execute potentially destructive operations safely from your phone:

  1. Phase 1 (readonly): Claude analyzes the situation using only read tools — no side effects
  2. Phase 2 (confirm): Review the analysis in Telegram, then tap [Execute] or [Cancel]
  3. Phase 3 (execute): Claude resumes the same session with full tool access
/task Refactor the error handling in claude-bridge.py

You can also reply to any message with /task to use that message as the task description.

Security Fix: Tool Profiles

The --allowedTools CLI flag does nothing in claude -p mode. Our readonly profile was completely ineffective — Claude had full tool access regardless of the setting.

Fix: Switched to --tools, which controls the actual available tool set. Tools not in the list are invisible to Claude.

Bug Fix: Connection Pool Exhaustion

Long-running claude -p calls could exhaust the default connection pool (size 1), causing the bot to silently stop responding. Fixed by configuring explicit pool sizes: 16 for main requests, 4 for polling.

Infrastructure: Sync Checklist

Added CHANGELOG.md and updated the internal sync checklist to cover external-facing files (README, CHANGELOG, config template) — ensuring documentation stays in sync with code changes.
