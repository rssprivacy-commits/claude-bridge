What is Claude Bridge?

Claude Bridge is a lightweight bridge service connecting Telegram to Claude Code CLI, letting you control Claude Code from anywhere via your phone. No extra servers, no API costs — just your Mac running Claude Code and a Telegram bot.

Key Features

  • Zero extra API cost — reuses your Claude Max subscription ($100/mo). The bridge is just I/O; all intelligence comes from claude -p
  • Multi-project management — switch between projects with /p. Each project gets its own CLAUDE.md, session state, and cost tracking
  • Session persistence — SQLite-backed sessions with --resume support. Pick up where you left off
  • Image support — send photos from Telegram, Claude reads them via the Read tool
  • Cost tracking — daily budget limits, per-project cost breakdown, 7-day history via /cost
  • InlineKeyboard UI — interactive buttons for project selection, model switching, tool permissions

Architecture Highlights

  • stdin pipe for messages — CLI args would choke on - prefixed Chinese text (parsed as flags). stdin solves this cleanly
  • Environment isolation — subprocess unsets CLAUDECODE and CLAUDE_PROJECT_DIR to prevent nested session detection
  • LaunchAgent with KeepAlive — automatic restart on crash, boot-time startup, throttle interval for stability
  • Bot Token in macOS Keychain — never stored in plaintext config files
  • Per-user FIFO lock + global semaphore — prevents session conflicts while limiting total concurrent workers

Tool Permission Profiles

Three built-in profiles switchable via /tools:

  • readonly (default) — Read, Grep, Glob, WebSearch, plus safe shell commands
  • standard — full Read/Write/Edit/Bash access
  • restricted — Read, Grep, Glob, WebSearch only
