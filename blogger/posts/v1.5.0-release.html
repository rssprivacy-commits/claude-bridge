Claude Bridge v1.5.0 brings architectural stability fixes, ElevenLabs account management, and smarter Telegram interactions.

Highlights

  • /el command — ElevenLabs account dashboard right in Telegram. Check credits usage, voice slots, clone capabilities, and next billing amount without logging into the web console.
  • 21 ElevenLabs voices — expanded from 5 to the full Creator voice library. Male and female voices with diverse styles, all selectable via /voice with live preview.
  • Smart model routing — short simple queries (<80 chars) auto-downgrade to Sonnet + low effort for faster responses. Complex tasks still route to your configured model.
  • Telegram behavior constraints — system context injection enforces concise replies, action-first style, and proper sensitive data handling.

Architecture Fixes

Three stability issues that caused intermittent errors in production have been fixed:

  • Task GC leak — fire-and-forget asyncio.create_task() calls (cron scheduler, agent loop) had no strong reference. GC could destroy pending tasks, causing "Task was destroyed" errors. New _create_background_task() holds references with auto-cleanup.
  • Result type safety — Claude CLI occasionally returns unexpected JSON types. All 7 invoke result chains now pass through _safe_result() normalization.
  • Stream buffer overflow — readline() with a 4MB limit still overflowed on very large tool results. Replaced with manual chunked reading — no size limit, no more "Separator/chunk exceed limit" crashes.

Security

  • Sensitive messages (passwords, tokens) auto-deleted from chat after processing
  • ElevenLabs account upgraded to Creator plan with 2FA enabled
