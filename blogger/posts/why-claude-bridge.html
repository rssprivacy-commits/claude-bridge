Why Claude Bridge? — A New Way to Control Your Mac with AI

If you're a Claude Max subscriber, you're paying $100/month for one of the most powerful AI coding assistants available. But there's a catch: you can only use it while sitting in front of your computer.

Claude Bridge changes that.

The Problem with Traditional Remote Control

Most remote access solutions require significant setup and have real limitations:

  • SSH — requires a public IP or port forwarding, exposes your machine to the internet
  • Tailscale — great tool, but in China it conflicts with VPN proxies (you can't run both smoothly)
  • VNC / TeamViewer — heavy, slow, blocked by GFW in China
  • Remote Desktop — requires the screen to be on, consumes resources

All of these solve the same problem the same way: give you a window into your computer. But that's not actually what most developers need.

What You Actually Need

Most of the time, you don't need to see your computer. You need to tell it to do something.

"Check if the service is still running."
"Look at the logs and tell me if there are any errors."
"Deploy the fix we discussed earlier."
"Run the tests and send me the results."

These are instructions, not mouse clicks. And Claude Code is already perfect at executing them — it just needs a way to receive them from anywhere.

That's exactly what Claude Bridge does.

How It Works

Your Phone (Telegram)
       ↓
  Claude Bridge Bot
       ↓
  Claude Code CLI  ←  reads your project's CLAUDE.md
       ↓
  Your Mac (local execution)
       ↓
  Result → back to Telegram

You send a message to your Telegram bot. Claude Bridge passes it to Claude Code running locally on your Mac. Claude Code reads, thinks, executes — then sends the result back to your phone.

No public IP. No port forwarding. No VPN conflicts. Just Telegram.

The China User Advantage

For users in China, this is particularly significant.

The typical setup is: VPN proxy (like mihomo/clash) for internet access + Tailscale for device connectivity. The problem? They conflict. Running both simultaneously causes routing issues, dropped connections, and unstable performance.

Claude Bridge sidesteps this entirely:

  • Telegram traffic goes through your existing proxy (mihomo port 1082)
  • Claude Code runs locally — no outbound connection needed
  • No Tailscale required at all
  • One proxy, one connection, everything works

As long as you can open Telegram, you have full access to Claude Code on your Mac.

A New Interaction Paradigm

Traditional remote control: you control the computer
Claude Bridge: you instruct an AI, which controls the computer

This is the difference between driving a car and telling a driver where to go. You don't need to know the exact commands. You don't need to remember the file paths. You just describe what you want.

  • ✅ "Check if OpenClaw is running and healthy"
  • ✅ "Find any Python errors in today's logs"
  • ✅ "Run the deployment script for the new feature"
  • ✅ "What's the current disk usage on my Mac?"

Claude Code figures out how. You get the answer on your phone.

Zero Extra Cost

If you already have Claude Max ($100/month), Claude Bridge costs nothing additional. It uses the same claude CLI that you're already paying for. The bridge itself is just I/O — all the intelligence comes from Claude Code.

No API keys. No separate billing. No usage limits beyond what your Max subscription provides.

Real Use Cases

Away from desk: You're in a meeting and remember you need to check if a cron job ran. Pull out your phone, ask the bot. Done in 30 seconds without leaving the room.

Traveling: You're in a different timezone and your automated system needs a fix. Open Telegram, describe the issue, Claude Code patches it while you're on the train.

Multi-project management: Register multiple projects with /p add. Switch between your personal infrastructure, work projects, and side projects — all from the same Telegram conversation.

Peace of mind: Before sleeping, ask "is everything running normally?" Get a quick health check without opening your laptop.
